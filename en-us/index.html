What is OpenKruise

Kruise is at the core of the OpenKruise project. It is a set of controllers which extends and complements Kubernetes core controllers on application workload management.

Feature List

  • Scalability

    Brokers, producers, consumers, and name servers adopt special deployment and processing methods that bring strong scale-out ability.

  • Distributed transaction

    Fescar implements a function similar to distributed transaction processing of X/Open XA, which allows multiple resources to be accessed within the same transaction.

  • Cache and Cache Maintenance

    Fescar makes the best use of system memory cache to maintain data to the file system through flushing either synchronously or asynchronously.

  • Message filtering

    Apache Fescar supports flexible syntax expressions to filter messages, which reduces transmission of useless messages to consumers.

  • Consumer offset

    Based on the message storage model of Apache Fescar, consumer offset can be reset by the time, accurate to a millisecond. Messages can be re-consumed from the earliest offset and the latest offset.

  • Timed messaging

    Apache Fescar supports timed messaging, but the time precision has specific levels.
